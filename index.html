PC 가상화 지원 확인

Docker가 동작하려면 CPU 가상화 기능이 필수입니다

+

Docker는 가상화 기술 위에서 동작합니다. CPU의 VT-x(Intel) / AMD-V(AMD) 기능이 활성화되어 있어야 설치가 가능합니다.

방법 1. 작업관리자로 확인 (가장 쉬움)

  • Ctrl + Shift + Esc → 작업관리자 실행
  • 상단 성능(Performance) 탭 클릭
  • 좌측 CPU 선택
  • 우측 하단 가상화(Virtualization) 항목 확인
📋 확인 결과 해석
  • Virtualization: Enabled → ✅ 활성화 완료. 바로 STEP 1으로
  • Virtualization: Disabled → ⚠️ BIOS에서 꺼진 상태. BIOS 진입 필요
  • Virtualization: Not supported → ❌ CPU가 지원 안 함. 다른 PC 필요

방법 2. CMD 명령어로 확인

CMD / PowerShell 
systeminfo | find "Hyper-V"

BIOS에서 가상화 활성화하기

위에서 Disabled가 나왔다면 다음 절차로 활성화합니다.

  • PC 부팅 시 Del 또는 F2 키를 연타하여 BIOS 진입
  • Advanced 또는 CPU Configuration 메뉴 진입
  • Intel CPU → Intel Virtualization Technology (VT-x) → Enabled
  • AMD CPU → SVM Mode 또는 AMD-V → Enabled
  • F10으로 저장 후 재부팅
💡 제조사별 BIOS 진입키
삼성·LG·ASUS: F2 · HP: F10 · Dell: F2 또는 F12 · Lenovo: F1 또는 F2 · 자작PC: Del
⚠️ 자주 막히는 지점
BIOS 메뉴 위치가 제조사마다 다릅니다. "Virtualization", "VT", "SVM" 같은 단어를 메뉴에서 검색해보세요. 저장(Save & Exit)을 잊으면 적용되지 않습니다.
01

Docker Desktop 설치

Windows용 Docker Desktop을 다운로드 및 설치합니다

+

공식 다운로드 링크

아래 링크에서 Docker Desktop for Windows (AMD64)를 다운로드합니다.

URL 
https://www.docker.com/get-started/

설치 단계

  • 다운로드한 Docker Desktop Installer.exe 실행
  • 설치 옵션 중 Use WSL 2 instead of Hyper-V 체크 (권장)
  • 설치 완료 후 PC 재부팅
  • 재부팅 후 Docker Desktop 자동 실행 → 약관 동의
  • "Engine running" 상태 확인
⚠️ Windows Home 사용자 주의
Windows 10/11 Home Edition은 Hyper-V를 지원하지 않습니다. WSL2 백엔드로 설치해야 합니다. 설치 마법사가 자동으로 안내해주며, WSL2가 없으면 자동 설치됩니다.

수동 설치가 필요하면 PowerShell(관리자)에서: wsl --install

설치 확인 (선택)

설치가 정상적인지 PowerShell에서 확인:

PowerShell 
docker --version
docker run hello-world

기본 설치 경로

PATH 
C:\Program Files\Docker\Docker
💡 RAM 8GB도 가능한가?
Docker Desktop 최소 요구사항은 4GB RAM이지만 8GB부터 안정적입니다. n8n + 워크플로우 1~2개는 8GB로 충분합니다. 무거운 워크플로우(LLM, 영상처리 등)는 16GB 권장.

n8n 컨테이너 생성

Docker Desktop에서 n8n 이미지를 받아 컨테이너를 실행합니다

+

이미지 검색 및 Pull

  • Docker Desktop 실행
  • 상단 검색창에서 n8n 검색
  • 첫 번째 결과 n8nio/n8n (공식 이미지) 선택
  • 우측 Pull 클릭 → 다운로드 대기 (수 분 소요)

컨테이너 실행 옵션 설정

이미지 다운로드 완료 후 Images 탭에서 n8nio/n8n 우측 Run 클릭. Optional settings 패널을 펼쳐 다음과 같이 설정합니다.

항목값설명
Container namen8n_buylife자유롭게 (영문+언더스코어)
Host port5678브라우저 접속 포트
Volume - Host pathD:\Docker\n8n본인 PC의 저장 경로 (사전 생성)
Volume - Container path/home/node/.n8nn8n 내부 데이터 경로 (고정)
Env VariableN8N_COMMUNITY_PACKAGES_ALLOW_TOOL_USAGE커뮤니티 노드 허용
Env Valuetrue활성화
⚠️ Volume Host path 주의
  • 해당 폴더는 사전에 직접 만들어야 합니다 (예: D:\Docker\n8n)
  • D드라이브가 없으면 → C:\Docker\n8n 등으로 변경
  • 이 폴더에 워크플로우·인증정보 데이터가 저장됩니다
  • 컨테이너 삭제해도 이 폴더만 있으면 데이터 보존

환경변수 정확한 값

Environment Variable 
# Variable
N8N_COMMUNITY_PACKAGES_ALLOW_TOOL_USAGE

# Value
true

실행 및 접속

  • 모든 설정 완료 후 우측 하단 Run 버튼 클릭
  • 컨테이너 상태가 Running으로 변경 확인
  • 브라우저에서 접속: http://localhost:5678
  • n8n 회원가입 화면 표시 → 이메일·비밀번호 입력
💡 License key는 어떻게?
가입 후 License key 입력 화면이 나오면 Skip(건너뛰기) 클릭하면 됩니다. Self-host 버전은 Community Edition으로 무료 사용 가능합니다 (Fair Code 라이선스).

커맨드라인 대안 (PowerShell)

GUI 대신 명령어로 동일한 컨테이너를 만들 수도 있습니다.

PowerShell 
docker run -d --name n8n_buylife ^
  -p 5678:5678 ^
  -v D:\Docker\n8n:/home/node/.n8n ^
  -e N8N_COMMUNITY_PACKAGES_ALLOW_TOOL_USAGE=true ^
  --restart unless-stopped ^
  n8nio/n8n:latest
03

Cloudflare 설치 & 인증서 발급

외부 도메인 연결을 위한 Cloudflare Tunnel 환경 구축

+

로컬 PC를 외부에서 접속 가능한 서버로 만들기 위해 Cloudflare Tunnel을 사용합니다. 공유기 포트포워딩이나 IP 노출 없이도 https://n8n.내도메인으로 접속할 수 있게 해주는 핵심 기술입니다.

사용자 → Cloudflare → 터널 → 내 PC의 n8n

3-1. Cloudflare 계정 가입

  • cloudflare.com 접속 → 무료 가입
  • 도메인을 Cloudflare에 등록해야 함 (DNS 이전)
  • 가비아·후이즈 등에서 산 도메인의 네임서버를 Cloudflare로 변경
  • DNS 전파 대기 (보통 수 분~수 시간)

3-2. cloudflared 설치 (Windows winget)

관리자 권한 PowerShell에서:

PowerShell (관리자) 
winget install --id Cloudflare.cloudflared

3-3. 설치 경로 확인

기본 설치 위치 
C:\Users\내계정\AppData\Local\Microsoft\WinGet\Packages\Cloudflare.cloudflared_Microsoft.Winget.Source_8wekyb3d8bbwe
📁 빠른 접근 방법
윈도우 탐색기 주소창에 다음을 입력하면 바로 이동: %LOCALAPPDATA%\Microsoft\WinGet\Packages\
AppData는 숨김 폴더입니다. 보기 → 숨긴 항목 체크 필요.

3-4. PATH 환경변수 등록

어디서나 cloudflared 명령을 쓸 수 있도록 등록. 관리자 권한 PowerShell:

PowerShell (관리자) · PATH 등록 
$machinePath = [Environment]::GetEnvironmentVariable("Path","Machine")

if ($machinePath -notlike "*D:\n8n-stack\cloudflared*") {
    [Environment]::SetEnvironmentVariable(
        "Path",
        $machinePath + ";D:\n8n-stack\cloudflared",
        "Machine"
    )
}
⚠️ 경로 수정 필수
D:\n8n-stack\cloudflared 부분을 본인 실제 설치 경로로 변경하세요. PATH 등록 후 PowerShell 재시작 해야 적용됩니다.

3-5. 설치 확인 (4가지 방법)

설치 확인 
# 1. 설치 경로 확인
where cloudflared

# 2. 버전 확인 (가장 추천)
cloudflared --version

# 3. PowerShell 인식 확인
Get-Command cloudflared -All

# 4. 실행 파일 직접 탐색
dir "C:\Program Files" -Recurse -Filter cloudflared.exe -ErrorAction SilentlyContinue

3-6. Cloudflare 인증서 발급

아래 명령어 실행 → 브라우저 자동 열림 → Cloudflare 로그인 → 도메인 선택 → 권한 부여:

인증서 발급 
cloudflared tunnel login
🔐 인증 흐름
① 명령 실행 → 브라우저 자동 열림
② Cloudflare 계정 로그인
③ 내 도메인 선택 (예: buyfile.xyz)
④ 권한 부여 클릭
⑤ 터미널에 "You have successfully logged in" 출력
⑥ cert.pem 인증서 파일 자동 생성 ✅

브라우저가 자동으로 안 열리면 터미널에 출력된 URL을 직접 복사해서 접속하세요.

3-7. 터널 생성 (이름 지정)

터널 생성 
cloudflared tunnel create n8n-tunnel

실행 결과 예시:

출력 예시 
Created tunnel n8n-tunnel with id d8769d83-9693-4fcb-aaee-e3632f896df3
Tunnel credentials written to:
C:\Users\내계정명\.cloudflared\d8769d83-9693-4fcb-aaee-e3632f896df3.json
🔑 중요 — 터널 UUID 메모!
출력되는 UUID(긴 영문숫자 조합)는 다음 단계에서 계속 사용합니다. 반드시 메모하세요. JSON 파일명이 곧 터널 ID입니다.

예시: d8769d83-9693-4fcb-aaee-e3632f896df3

3-8. config.yml 환경설정 파일 생성

아래 경로에 config.yml 파일을 만들고 다음 내용을 입력:

저장 위치 
C:\Users\내계정명\.cloudflared\config.yml
config.yml 내용 
tunnel: d8769d83-9693-4fcb-aaee-e3632f896df3
credentials-file: D:/n8n-stack/cloudflared/d8769d83-9693-4fcb-aaee-e3632f896df3.json

ingress:
  - hostname: n8n.내도메인
    service: http://localhost:5678
    originRequest:
      httpHostHeader: n8n.내도메인
      noTLSVerify: true
  - service: http_status:404
⚠️ YAML 작성 주의사항
  • tunnel UUID → 본인 터널 UUID로 변경
  • credentials-file 경로 → 본인 JSON 파일 실제 경로
  • n8n.내도메인 → 본인 도메인 (예: n8n.buylife.xyz)
  • 들여쓰기(공백 2칸) 정확히 지키기 — YAML은 들여쓰기 민감
  • 메모장 저장 시 UTF-8 인코딩, 인코딩 없음(BOM 제외)
04

Docker Desktop 자동 시작 설정

PC 부팅 시 Docker가 자동으로 실행되어 n8n이 24/7 동작

+

PC를 끄거나 재부팅한 후에도 n8n이 자동으로 켜지게 하려면 Docker가 부팅 시 자동 실행되어야 합니다.

설정 경로

  1. Docker Desktop 실행
  2. 우측 상단 ⚙️ (Settings) 클릭
  3. 좌측 메뉴에서 General 선택
  4. 상단 두 체크박스 모두 ✅ 체크
  5. 우측 하단 Apply 버튼 클릭
옵션설정의미
Start Docker Desktop when you sign in to your computer✅ 체크PC 부팅 시 Docker 자동 실행
Open Docker Dashboard when Docker Desktop starts✅ 체크Docker 시작 시 대시보드 자동 열기
🔄 컨테이너 자동 재시작 정책
Docker가 자동 시작되어도 n8n 컨테이너 자체는 별도 설정이 필요합니다. STEP 2에서 컨테이너 생성 시 restart 정책을 unless-stopped로 지정하면 PC 재부팅 후에도 컨테이너가 자동 실행됩니다.

docker-compose.yml을 사용하는 경우 (STEP 6 참조): restart: unless-stopped
⚠️ Apply 버튼 클릭 필수
체크박스만 체크하고 Apply 버튼을 누르지 않으면 저장되지 않습니다. 설정 후 PC 재부팅으로 자동 실행되는지 확인하세요.
05

Windows 절전 기능 완전 해제

24/7 운영을 위해 절전·디스플레이 꺼짐·네트워크 절전을 모두 비활성화

+

PC가 절전모드로 들어가면 n8n 서버도 멈춥니다. 24시간 운영을 위해 3가지 절전 기능을 모두 해제해야 합니다.

방법 1. Windows 설정 GUI

  1. 설정 → 시스템 → 전원 진입
  2. 절전 모드 → 항상 절전 모드 사용: 끔
  3. 화면, 절전 모드, 최대 절전 모드 시간 제한 → 모두 없음

방법 2. PowerShell 명령어 (한 번에 처리)

관리자 권한 PowerShell에서:

PowerShell (관리자) 
# 최대 절전 모드 OFF
powercfg /hibernate off

# 절전 모드 시간 제한 해제 (전원 연결 시)
powercfg /change standby-timeout-ac 0

# 모니터 꺼짐 시간 해제
powercfg /change monitor-timeout-ac 0

# 디스크 절전 시간 해제
powercfg /change disk-timeout-ac 0

방법 3. 네트워크 어댑터 절전 해제 ⭐ 중요!

  4. 전원 관리 탭 선택
  5. 전원을 절약하기 위해 컴퓨터가 이 장치를 끌 수 있음 체크 해제
  6. 확인 클릭

폴더 구조 생성 & PATH 등록

n8n-stack 작업 폴더를 만들고 인증서 환경변수를 등록합니다

+

전체 폴더 구조

다음 구조로 폴더와 파일을 정리합니다.

바우하우스 폴더 구조도 · D:\n8n-stack 기준

6-1. 폴더 만들기

탐색기에서 또는 PowerShell로 다음 폴더를 생성:

PowerShell 
mkdir D:\Docker
mkdir D:\n8n-data
mkdir D:\n8n-stack
mkdir D:\n8n-stack\cloudflared

6-2. cert.pem 인증서 파일 복사

STEP 3에서 생성된 cert.pem을 cloudflared 폴더로 복사:

PowerShell 
Copy-Item "$env:USERPROFILE\.cloudflared\cert.pem" -Destination "D:\n8n-stack\cloudflared\"

6-3. JSON 자격증명 파일 복사

STEP 3에서 생성된 터널 UUID JSON 파일도 복사:

PowerShell 
Copy-Item "$env:USERPROFILE\.cloudflared\*.json" -Destination "D:\n8n-stack\cloudflared\"

6-4. config.yml 파일 작성

STEP 3에서 만든 config.yml도 D:\n8n-stack\cloudflared\에 저장. 내용은 STEP 3-8 참조.

6-5. TUNNEL_ORIGIN_CERT 환경변수 등록

관리자 권한 PowerShell에서:

PowerShell (관리자) · 환경변수 등록 
setx TUNNEL_ORIGIN_CERT "D:\n8n-stack\cloudflared\cert.pem" /M
옵션설명
setx환경변수 영구 등록 (재부팅 후에도 유지)
TUNNEL_ORIGIN_CERTCloudflare가 인식하는 고정 변수명 (변경 불가)
/MMachine(시스템 전체) 적용 · 관리자 권한 필요
✅ 환경변수 등록 확인
새 PowerShell 창을 열고 다음을 입력해 확인:
echo $env:TUNNEL_ORIGIN_CERT
경로가 출력되면 성공 ✅

터널 실행 & DNS 라우팅 연결

Cloudflare 터널을 실행하고 도메인을 연결합니다

+

7-1. cloudflared 폴더로 이동

PowerShell 
cd D:\n8n-stack\cloudflared

7-2. 터널 실행

두 가지 방법 중 선택:

방법 A · 터널 이름으로 실행 
cloudflared tunnel run n8n-tunnel
방법 B · config 파일 지정 실행 
cloudflared tunnel --config config.yml run
📌 터미널 닫지 마세요!
tunnel run은 포그라운드 실행입니다. 이 터미널 창을 닫으면 터널이 종료됩니다. 24/7 운영하려면 Windows 서비스로 등록해야 합니다 (부록 참조). 일단은 새 PowerShell 창을 하나 더 열어 7-3부터 진행하세요.

7-3. 터널 목록 확인

새 PowerShell 창 
cloudflared tunnel list

출력 목록에서 n8n-tunnel이 HEALTHY 상태인지 확인.

7-4. DNS 라우트 연결

도메인을 터널에 연결합니다:

DNS 라우트 등록 
cloudflared tunnel route dns n8n-tunnel n8n.내도메인
구성 요소설명예시
route dnsDNS 라우팅 등록 명령(고정)
n8n-tunnel연결할 터널 이름STEP 3-7에서 생성한 이름
n8n.내도메인외부 접속할 서브도메인n8n.buyfile.xyz
🌐 DNS 라우트 등록 후
Cloudflare 대시보드(dash.cloudflare.com) → 내 도메인 → DNS 탭에서 CNAME 레코드가 자동 생성된 것을 확인할 수 있습니다.
n8n → [터널UUID].cfargotunnel.com

부록 · n8n 업데이트 방법 ⭐ 매우 중요

n8n을 최신 버전으로 업데이트하는 가장 안전한 방법

+

n8n은 거의 매주 새 버전이 나옵니다. 새 기능과 보안 패치를 받으려면 주기적 업데이트가 필요합니다.

업데이트 핵심 4줄 명령어

⭐ 가장 자주 쓰는 명령어 세트 
# 1. n8n-stack 폴더로 이동
cd D:\n8n-stack

# 2. 컨테이너 중지 및 제거
docker compose down

# 3. 컨테이너 재시작 (자동 pull)
docker compose up -d

# 4. 상태 확인
docker compose ps

전체 업데이트 흐름

  1. npm view n8n version → 최신 버전 확인
  2. docker-compose.yml 열기
  3. image: n8nio/n8n:1.x.x 부분의 버전 수정 또는 :latest 유지
  4. cd D:\n8n-stack
  5. docker compose down → 기존 컨테이너 중지
  6. docker compose up -d → 새 버전으로 자동 pull & 재시작
  7. docker compose ps → 정상 실행 확인 ✅

최신 버전 확인

버전 확인 
npm view n8n version

컨테이너 내 버전

docker compose logs n8n --tail 100

자주 묻는 질문 & 트러블슈팅

강의 현장에서 실제로 나온 질문과 해결책 모음. 각 질문을 클릭하면 답변이 펼쳐집니다.

💻 환경 / 사양

Q RAM 8GB로도 n8n 운영이 가능한가요? +

가능합니다. Docker Desktop의 최소 요구사항은 4GB이며, n8n 단일 컨테이너는 통상 500MB~1.5GB 정도 사용합니다.

  • 8GB: n8n + 워크플로우 1~3개 동시 실행 가능
  • 16GB: 무거운 워크플로우 (LLM, 영상처리, 대량 데이터) 권장
  • 32GB: 다중 서비스(n8n + PostgreSQL + Redis + 로컬 AI 모델) 운영용

💡 도시아재 운영 구성: Mac mini M4 24GB / 512GB로 Hermes 로컬 LLM까지 함께 운영 검토 중.

Q D드라이브가 없는 PC인데 어떻게 하나요? +

C드라이브에 그대로 만들면 됩니다. 가이드의 모든 D:\ 경로를 C:\로 바꾸세요.

  • D:\Docker → C:\Docker
  • D:\n8n-stack → C:\n8n-stack
  • config.yml 내부 경로도 모두 C로 변경 필수
  • setx TUNNEL_ORIGIN_CERT 명령어 경로도 C로

⚠️ 주의: C드라이브 용량이 부족하면 Docker 이미지가 쌓이면서 오류 발생. 최소 30GB 여유 확보를 권장합니다.

Q Windows 10 Home에서도 가능한가요? +

가능합니다. 다만 Pro 버전과 다른 점이 있습니다.

  • Home Edition은 Hyper-V 미지원 → WSL2 백엔드로 자동 설치
  • Docker Desktop 설치 마법사가 자동으로 WSL2 설치 옵션 제공
  • 설치 안 되면 PowerShell(관리자)에서: wsl --install
  • WSL2 설치 후 PC 재부팅 필요

참고: Hyper-V가 켜져 있으면 다른 가상화 도구(VMware, VirtualBox)와 충돌할 수 있으므로 Home Edition + WSL2 조합이 오히려 호환성 좋음.

⚠️ 설치 오류

Q cloudflared 명령어가 인식되지 않습니다 ("cloudflared'은(는) 인식되지 않습니다") +

PATH 환경변수가 등록되지 않았거나, PowerShell이 재시작되지 않은 경우입니다.

해결 1. 실행 파일 위치 직접 확인:

where.exe cloudflared

해결 2. PATH에 등록 후 PowerShell 완전 재시작 (모든 창 닫고 새로 열기).

해결 3. 절대경로로 직접 실행:

"C:\Program Files\Cloudflare\cloudflared\cloudflared.exe" --version

💡 도시아재 사례: uvx 인식 안 될 때 동일하게 절대경로로 해결했음 (C:\Python314\Scripts\uvx.exe).

Q Docker Desktop이 시작되지 않습니다 ("Docker daemon not running") +

가장 흔한 원인 4가지를 순서대로 확인하세요.

  1. 가상화 활성화 확인 → 작업관리자 → 성능 → CPU → Virtualization: Enabled
  2. WSL2 설치 상태 확인 → PowerShell에서 wsl -l -v → 버전 2가 표시되어야 함
  3. Hyper-V 충돌 → VirtualBox/VMware 사용 중이면 일시 중지
  4. Docker Desktop 재시작 → 트레이 아이콘 우클릭 → Restart Docker

그래도 안 되면 Docker Desktop 완전 재설치 (설정·데이터 보존 옵션).

Q n8n 컨테이너가 자꾸 멈춥니다 (Exited 상태) +

로그를 보고 원인 파악이 우선입니다.

docker logs n8n_buylife --tail 200

흔한 원인 4가지:

  • 볼륨 경로 오류: Host path 폴더가 없거나 권한 부족 → 폴더 직접 생성
  • 포트 충돌: 5678 포트가 이미 사용 중 → netstat -ano | findstr 5678 확인
  • 메모리 부족: Docker Desktop 메모리 할당 부족 → Settings → Resources에서 증가
  • 이미지 손상: docker compose down 후 docker compose pull 후 재시작

🌐 도메인 / 터널

Q 도메인 접속 시 404 오류가 납니다 +

config.yml의 ingress 설정에 문제가 있을 가능성이 큽니다.

체크리스트:

  • config.yml의 hostname이 실제 등록한 도메인과 정확히 일치하는가
  • service: http://localhost:5678 포트 번호가 정확한가
  • 마지막 ingress 항목이 service: http_status:404로 끝나는가 (필수)
  • 들여쓰기(공백 2칸)가 정확한가 (YAML은 들여쓰기 민감)
  • 터널을 실행한 후 config 파일을 수정했다면 터널 재시작 필수

config.yml 수정 후:

# 기존 터널 종료 (Ctrl+C)
cloudflared tunnel --config config.yml run
Q 도메인 접속 시 502 Bad Gateway 오류 +

터널은 살아있지만 n8n 컨테이너가 응답하지 않는 상태입니다.

  1. docker ps → n8n 컨테이너 Up 상태인지
  2. curl http://localhost:5678 → 로컬에서 직접 접속 가능한지
  3. 안 되면 docker restart n8n_buylife
  4. 그래도 안 되면 컨테이너 로그 확인: docker logs n8n_buylife
Q SSL 인증서 오류 (NET::ERR_CERT_AUTHORITY_INVALID) +

Cloudflare SSL 설정을 확인해야 합니다.

  1. Cloudflare 대시보드 접속
  2. 해당 도메인 선택
  3. 좌측 메뉴 SSL/TLS 클릭
  4. 암호화 모드를 Full 또는 Flexible로 설정
  5. 오류 지속 시 → Edge Certificates 탭 → Always Use HTTPS ON

💡 SSL 변경 후 적용까지 1~5분 소요. 브라우저 캐시 비우고 시크릿 모드 테스트.

Q 터널이 자꾸 끊깁니다. PC 켜둬도 외부 접속이 안 돼요. +

주요 원인 3가지:

  • 터미널 종료: cloudflared tunnel run은 터미널 종료 시 함께 종료됨 → Windows 서비스로 등록
  • 네트워크 어댑터 절전: STEP 5의 장치 관리자 설정 확인
  • 인터넷 연결 끊김: 와이파이보다는 유선 이더넷 권장

Windows 서비스로 등록 (관리자 PowerShell):

cloudflared service install

이렇게 등록하면 PC 부팅 시 자동 실행되고, 터미널 종료와 무관하게 24/7 동작합니다.

🔧 운영 / 보안

Q n8n에 license key를 꼭 입력해야 하나요? +

아니요. 건너뛰어도 됩니다.

  • n8n Self-host는 Community Edition (무료)으로 사용 가능 — Sustainable Use License
  • 회원가입 후 License key 입력 화면에서 Skip 클릭
  • 대부분의 기능(워크플로우 무제한, 모든 노드, AI 노드)을 무료로 사용
  • Business/Enterprise 기능(SSO, RBAC, 환경 분리, 외부 시크릿)은 유료 라이선스 필요

공인중개사 1인 사무실 운영에는 Community Edition이 충분합니다.

Q 워크플로우를 백업하려면 어떻게 하나요? +

3가지 방법이 있습니다.

방법 1. n8n UI에서 개별 Export

  • 워크플로우 우측 상단 ⋮ (점 세 개) 클릭
  • Download 선택 → JSON 파일 저장

방법 2. 볼륨 폴더 통째로 백업 ⭐ 추천

xcopy /E /I /Y D:\Docker\n8n D:\backup\n8n_2026-05-22

방법 3. CLI Export (전체 워크플로우)

docker exec -it n8n_buylife n8n export:workflow --all --output=/home/node/.n8n/backup.json

방법 2가 가장 안전합니다. 워크플로우 + Credential + 실행이력까지 모두 포함됩니다.

Q 외부에 노출했는데 보안이 걱정됩니다. 무엇을 해야 하나요? +

최소한 다음 5가지는 적용하세요.

  1. 강력한 비밀번호: n8n 회원가입 시 16자 이상, 특수문자 포함
  2. 2FA 활성화: Settings → My account → Enable 2FA
  3. Cloudflare Access: Zero Trust → Access로 IP/이메일 기반 접근 제한
  4. Webhook URL 노출 주의: Webhook은 URL만 알면 누구나 호출 가능 → 인증 토큰 추가
  5. 정기 업데이트: 보안 패치 자주 나오므로 주 1회 docker compose 업데이트

💡 Cloudflare Access 추천: 추가 비용 없이 사용 가능하며, 특정 이메일만 접속 허용/허용된 IP만 접속하도록 설정 가능. 사실상 가장 강력한 보안 추가 수단.

Q 한국 시간대(KST)로 워크플로우가 동작하게 하려면? +

n8n 컨테이너의 환경변수 2개를 추가하면 됩니다.

GENERIC_TIMEZONE=Asia/Seoul
TZ=Asia/Seoul

적용 방법:

  • Docker Desktop GUI → 컨테이너 설정 → Environment 추가 (재생성 필요)
  • 또는 docker-compose.yml의 environment 섹션에 추가
  • 적용 후 docker compose down && docker compose up -d
  • n8n 내부에서 Schedule Trigger 등이 한국시간 기준 동작
Q n8n Cloud 24€/월과 비교해서 정말 무료인가요? +

n8n 자체는 무료입니다. 단, 실제 운영비를 정확히 계산하면:

항목비용설명
n8n 라이선스무료Community Edition
Cloudflare Tunnel무료개인 사용 무료

API 사용료 (선택)워크플로우별OpenAI/Claude 등은 별도

n8n Cloud의 월 24€(약 36,000원)와 비교했을 때, 전기료를 감안해도 손익분기점은 매우 빠릅니다. 게다가 워크플로우 실행 횟수 제한이 없어 장기적으로 압도적 경제성을 갖습니다.
